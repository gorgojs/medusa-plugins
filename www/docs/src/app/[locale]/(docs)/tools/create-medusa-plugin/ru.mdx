# {frontmatter.title}

CLI-инструмент для быстрого создания плагина Medusa и примеров проектов в структуре монорепозитория, готовой к разработке.

## Использование

`create-medusa-plugin` использует официальный [`create-medusa-app`](https://docs.medusajs.com/resources/create-medusa-app) для создания проектов Medusa, включая папки плагина, backend и storefront. Благодаря этому итоговая структура соответствует официальным рекомендациям и остается полностью совместимой с экосистемой Medusa, так как основана на официальном генераторе приложений Medusa.

```bash
npx create-medusa-plugin [plugin-name] [options]
```

Замените `[plugin-name]` на желаемое имя плагина.

### Опции

| Опция                      | Описание                                                                                              | По умолчанию      |
| -------------------------- | ----------------------------------------------------------------------------------------------------- | ----------------- |
| `-r, --repo <name>`        | Имя корневого репозитория (монорепозитория)                                                           | _не задано_       |
| `-e, --example <name>`     | Имя папки с примером, в которой будут созданы примеры `medusa` и `medusa-storefront`                 | _не задано_       |
| `--directory-path <path>`  | Путь к директории, в которой будет создан корневой репозиторий                                        | Текущая директория |
| `--plugin-repo-url <url>`  | URL репозитория для создания плагина Medusa (передает `repo-url` в `create-medusa-app`)              | _не задано_       |
| `--example-repo-url <url>` | URL репозитория для создания примера backend на Medusa (передает `repo-url` в `create-medusa-app`)   | _не задано_       |
| `--no-storefront`          | Не включать пример storefront (передается как `--with-nextjs-starter=false` в `create-medusa-app`)   | `false`           |
| `--skip-example-db`        | Пропустить создание базы данных для примера Medusa (передает `--skip-db` в `create-medusa-app`)      | `false`           |
| `--verbose`                | Показывать все логи внутренних команд (передает `--verbose` в `create-medusa-app`)                   | `false`           |

## Примеры

* **Базовое использование**

  ```bash
  npx create-medusa-plugin
  ```

  Эта команда в интерактивном режиме запросит имя плагина и другие параметры конфигурации.

* **С опциями**

  ```bash
  npx create-medusa-plugin my-plugin -r my-plugin-monorepo -e my-plugin-example --skip-example-db
  ```

  Эта команда создаст следующую структуру монорепозитория и установит в нее проекты Medusa:

  ```
  /
  └── my-plugin-monorepo/
      ├── examples/
      │   └── my-plugin-example/
      │      ├── medusa/
      │      └── medusa-storefront/
      ├── packages/
      │   └── my-plugin/
      ├── package.json
      └── README.md
  ```

* **С областью имен плагина**

  ```bash
  npx create-medusa-plugin @my-scope/my-plugin -r my-plugin-monorepo -e my-plugin-example --skip-example-db
  ```

  Эта команда создаст следующую структуру монорепозитория, поместит scoped-плагин во вложенную папку внутри `packages` и установит в нее проекты Medusa:


  ```
  /
  └── my-plugin-monorepo/
      ├── examples/
      │   └── my-plugin-example/
      │      ├── medusa/
      │      └── medusa-storefront/
      ├── packages/
      │   └── @my-scope/
      │      └── my-plugin/
      ├── package.json
      └── README.md
  ```
