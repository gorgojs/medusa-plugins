# Телеметрия

Все плагины `gorgo` собирают анонимные данные об использовании, чтобы понять, какие функции востребованы и в каких окружениях работают плагины. Это помогает нам расставлять приоритеты в разработке на основе реальной картины, а не догадок. Персональные данные, данные клиентов и бизнес-данные никогда не собираются.

---

## Что собирается

Каждое событие содержит следующие поля:

| Поле | Значение |
|---|---|
| `machine_id` | Случайный идентификатор, сгенерированный при первом запуске и сохранённый в `~/.gorgo/telemetry.json`. Формируется из случайных байт — не связан с железом, именем хоста или любым системным идентификатором. |
| `session_id` | UUID, генерируется заново при каждом запуске процесса Medusa. |
| `plugin.name` | Имя npm-пакета плагина (например, `@gorgo/medusa-payment-robokassa`). |
| `plugin.version` | Установленная версия плагина. |
| `env.medusa_version` | Версия `@medusajs/medusa` в проекте. |
| `env.node_version` | Версия Node.js. |
| `env.os` | Операционная система: `linux`, `darwin` или `win32`. |
| `env.arch` | Архитектура процессора: `x64` или `arm64`. |
| `env.ci` | `true`, если обнаружена CI-среда (GitHub Actions, GitLab CI, CircleCI и др.). |
| `env.docker` | `true`, если запуск происходит внутри Docker-контейнера. |
| `env.node_env` | Значение переменной `NODE_ENV` (`production`, `development`, `test`). |
| `env.locale` | Локаль системы (например, `ru_RU.UTF-8`), читается из переменных окружения shell. |
| `env.timezone` | Часовой пояс системы (например, `Europe/Moscow`). |
| `env.package_manager` | Используемый пакетный менеджер: `npm`, `yarn`, `pnpm` или `unknown`. |

Некоторые события дополнительно содержат объект `properties` со специфичными для этого события полями. Полный список описан в [каталоге событий](#события) ниже.

---

## Что никогда не собирается

- **Данные клиентов и магазина** — никаких ID заказов, названий товаров, SKU, цен, email или телефонов покупателей, любых данных из вашего магазина.
- **Секреты и учётные данные** — никаких API-ключей, токенов, паролей или переменных окружения вашего проекта.
- **IP-адреса** — коллектор получает ваш IP как часть HTTP-запроса, но не сохраняет и не логирует его.
- **Stack trace и тексты ошибок** — события об ошибках содержат только короткую классификацию (`reason` или `error_code`), но никогда сырой текст исключения.
- **Пути к файлам и директориям** вашего проекта.
- **Любые данные, позволяющие идентифицировать вас или ваш магазин.**

## Как отключить

Установите переменную окружения в значение `1` или `true`:

```bash
GORGO_DISABLE_TELEMETRY=true
```

Также можно отключить телеметрию вручную, отредактировав файл `~/.gorgo/telemetry.json`:

```json
{
  "enabled": false
}
```

При наличии переменной окружения телеметрия отключается автоматически. Файловый вариант удобен для окружений, где управление переменными окружения затруднено.

**CI-окружения:** в CI телеметрия работает, но уведомление о первом запуске (выводится в stdout при первом старте) подавляется, если установлена переменная `CI`.

## Исходный код

Этот модуль полностью открыт. Вся реализация доступна в [`packages/utils/telemetry`](https://github.com/gorgojs/medusa-plugins/tree/main/packages/utils/telemetry) в монорепозитории.

Все события отправляются на `https://telemetry.gorgojs.com/batch`.
